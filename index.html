UML类与类之间关系图

2019-04-15

UML类与类之间关系图

因为平时偶尔在设计的时候会用到时序图,还有类图,于是抽空学了下类图
网上看了下一个比较容易理解的图片,转自码匠笔记

  • 实现关系:可计算处理器和电脑,通常在程序里面类的关系是继承抽象类。
  • 泛化关系:电脑和mac,通常在程序里面类的关系是继承于非抽象类。
  • 组合关系:公司和部门,表示整体由部分组成,如果整体不存在则部分不存在。
  • 聚合关系:部门和码农,表示整体和部分的关系,如果整体不存在,部分依旧存在。
  • 关联关系:手机跟码农,手机是码农的一部分,不会因为他上班是否而变化关系,通常在程序里面表现为类变量。
  • 依赖关系:码农和mac,通常情况下是在程序里面通过构造函数、形参等体现。

导出大数据量excel

2019-04-01

背景:随着报表数据的增多,全表导出造成OOM,需要对现有系统的导出excel功能做进一步优化.

现状:通过查询出所有数据data,再查询出对应的需要转换的字段的字典,对data进行for循环处理数据,采用的是poi的XSSFworkbook,再对data数据for循环处理写到workbook。

优化:

  • 对data进行了太多次for循环处理,而且一次性加载太多数据容易造成OOM,mysql本身支持流式查询,上网查了下mybatis的流式处理可以使用ResultHandler来进行实现。多次的for循环处理数据可以直接在实现handleResult()方法中进行优化处理。
  • poi的XSSFworkbook支持的数据量不大,网上支持大数据的大都通过SXSSFworkbook,在new的时候指定内存保留的数据条数,可以避免造成OOM,而且也可以把写到workbook的操作也写到handleResult中。

总结:从原先的三次for循环变成一次for循环,采用SXSSFworkbook支持了大数据量的导出,避免的大数据量导出过程的OOM,通过减少对数据的多次处理,大大减少了导出时间。

核心代码

handleResult方法是实现了ResultHandler接口的方法,相当于在查询data的时候会进行调用处理,DataConverter是数据处理类,在此基础可以有UserDataConverter;DataWriter是数据写入类,在此基础可以有ExcelDataWriter.

MySQL的查询过程

2018-09-01

MySQL分为Server层和存储引擎层两部分

  • Server层包括连接器、查询缓存（一般没用）、分析器、优化器、执行器，很多功能都在这一层上面实现，mysql的核心功能、内置函数、跨引擎的功能等。
  • 存储引擎主要负责数据的存储和提取，主要有InnoDB、MyISAM、Memory等多种引擎。最常用的是InnoDB。

查询语句的查询过程

如:select * from user where id = 10;

  1. 先连接数据库，这时候连接器负责与客户端建立连接、获取权限、维持和管理连接。
  2. 连接建立完成后，会先到查询缓存看看，之前是否执行过这条语句，执行语句和结果是以key-value的形式缓存在内存中。（因为缓存往往利大于弊，所以不建议开启，8.0版本之后整块缓存功能已经删除。）
  3. 第三步是分析器对SQL进行解析，比如对select解析出是查询语句id解析出是列名。一般sql写错的时候会报错误，而且会第一个出现错误的位置。
  4. 经过分析器之后，在开始执行sql之前，会经过优化器，优化器是在表里面有多个索引的时候，决定使用哪个索引、或者是夺标关联查询join的时候，决定各个表的连接顺序。
  5. 优化器之后，语句的执行方案就出来了，就进入执行器阶段。开始执行的时候，先判断你是否对表有没有执行查询的权限，如果没有就返回没有权限的错误。如果有权限，就打开表继续执行，执行器会根据表的引擎定义，去使用这个引擎提供的接口。
  • 如果没索引则取第一行判断是否id是否等于10，如果不是则循环吓一跳，是则存入结果集然后执行下一行，直到表的最后一行。
  • 对于有索引的，执行逻辑类似，取满足条件的第一行的接口，之后循环。
  • 使用过explain的就看到过一个rows_examined的字段，表示执行过程扫描了多少行，这个值就是执行器每次调用引擎获取数据行的时候累加的。

算法之插入排序

2018-07-20

  1. 思想：插入排序首先是将第一位数看成是已经排序号的数列，第二位与之比较插入到合适的位置，这时候一二位就变成了已经排序的数组，接着第三位拿来和第二位比较，依次类推，直到排序完成。

  2. 时间复杂度分析，最坏情况下是全部逆序，插入第一个需要考虑一个元素，插入第二个需要考虑两个元素，最坏情况下是1+2+……+(n-1）,等差数列求和就是n^2 /2;O( n^2 );最好情况是O(n),就是只考虑一个元素;平均是O( n^2 );空间复杂度是O(1)。

算法之冒泡排序

2018-07-14

1.冒泡排序：简而言之，从最底部开始比较相邻的两个数的值大小，如果大的在底部，则交换。大的永远得在上面，这样最大的永远在上面，以此类推。

2.冒泡排序的最差时间分析是O( n^2 ),平均时间复杂度O( n^2 ),稳定，空间复杂度是O(1)。
